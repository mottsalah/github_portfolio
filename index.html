About Me

Hello and welcome to my portfolio! Here's a little about me and how I got into tech. I graduated with a degree in Biochemistry in 2023 at the University of Nottingham, but I was always curious about the tech industry. It started during secondary school when I realised I can inspect an element on a website to edit a page temporarily, and I used that to play jokes on my siblings, changing words and adding images of them to their school's website. Unfortunately, I wasn't very exposed to the tech industry since. That continued until my first bioinformatics module in my bachelor's degree. It was during this module that I discovered the power of technology and that sparked my interest. I didn't think to make a career out of it at that point yet however, until I started my final year dissertation, which was a quantitative analysis. We were tasked with counting the number of cancerous cells in a dataset based on a set of criteria, and I found this process very long and ineffiecient, so I thought "there must be a better way to do this". I ended up finding a way to train a classifier to count the cells for me, which ended up being both more effiecient, and more accurate than the method I'd used before. From there I was convinced, and my curiosity was through the roof, leading me to explore languages like python in my free time and starting my master's in Computer Science.

Software Engineering Group Project

The Software Engineering Group Project focused on designing a comprehensive system for the "BookMeIn2" application to manage business events effectively, adressing the needs of delegates, exhibitors, administrators, moderators, and speakers. We utilised Agile methodology, used AzureDevOps to create a retrospective board, and focused on a mobile-first design, creating mockups, UML and ERD diagrams, wireframes and detailed flow diagrams, all supported by clear documentation of team structures, design decisions, and requirements analysis. The final submission demonstrated how the designs aligned with functional and non-functional requirements, providing seamless solutions for event management across various stages. Design and Implementation of a web-based application

The "Friendzone" project involved designing and implementing a simple, secure, responsive social media platform inspired by Facebook. The design phase included an Entity Relationship Diagram (ERD), site map, database schema, and security analysis to address risks like SQL injection and XSS. The implementation phase delivered a functional prototype using PHP, JavaScript, HTML5, CSS3, and MySQL, featuring secure login, user profiles, post creation, commenting, and responsive interfaces for mobile and desktop. The project adhered to usability, accessibility, and security standards, ensuring a seamless and secure user experience. The project was aimed functionality rather than visual design, developing a fully functional prototype. The images show the "Home" and "My Profile" sections for both the desktop and mobile versions.

Library Reservation System

This project involved designing and implementing a library notification system, using python to manage book loans, returns, reservations, and membership cards. I focused on making the system scalable by using object-oriented programming and design patterns, which made it easy to add new types of notifications in the future. The system allowed for borrowing and returning books, issuing membership cards with unique IDs, and notifying users about late returns, fines, or reserved books becoming available. Data was stored and processed in JSON format, with CSV-to-JSON conversion included to handle data updates efficiently. The final result was a practical and flexible solution for managing library notifications. The images show some of the code and results.
